# 🤖 常见通讯频道机器人配置 SOP

如果你在运行 `setup.sh` 安装 OpenClaw 时选择了通讯频道，但不知道怎么获取对应的 **Webhook URL** 或 **Token / Secret**，请对照下方的图文步骤操作。

### 🔑 踩坑必看：没有企业/团队，怎么在飞书或钉钉建机器人？
飞书和钉钉的开发者后台**仅对有归属组织的用户开放**。如果你只是个人使用者，请先在手机端**免费创建一个空组织**作为测试测试场：
- **飞书**：打开手机飞书 APP -> 点击左上角个人头像 -> 点击**【创建或加入团队】** -> 选择**【创建团队】** -> 随便填一个名称（如“个人测试团队”）即可。
- **钉钉**：打开手机钉钉 APP -> 底部切换到**【通讯录】** -> 一直刷到最底端，点击**【创建/加入企业】** -> 随便填一个名称创建测试组织即可。

组建完成后，使用相同的手机号登录电脑端网页版的开发者后台，即可拥有全部机器人配置权限！

## 📱 1. 微信 (WeChat)
**微信不需要提前配置任何参数。**
- 在安装结束并提示容器启动后，查看日志或执行 `docker logs openclaw-main -f`。
- 终端会打印出一个“微信登录二维码”。
- 使用微信扫码授权登录即可开始对话。
- **注意**：网页版微信登录态由 `browser_profile` 挂载保留，即使之后重装 OpenClaw 也**不需要重新扫码**。

## 🔷 2. 钉钉 (DingTalk)
> 钉钉群里面的“自定义机器人 Webhook”只能单向发送消息，不能接收你说话的内容。要实现自由对话的新一代 Agent，必须创建【企业内部应用机器人】。

1. **进入钉钉开发者后台**：访问 [open.dingtalk.com](https://open.dingtalk.com)，登录具有对应企业权限的账号。
2. **创建应用**：点击顶部【应用开发】 -> 【企业内部开发】 -> 【机器人】 -> 点击【创建应用】。
3. **获取凭证**：创建成功后，在侧边栏【凭证与基础信息】中找到 **`AppKey`** 和 **`AppSecret`**。
4. **填入脚本**：将它们分别复制，粘贴到安装向导提示的对应位置。
5. **后续配置（容器启动后按需）**：
   - 在后台侧边栏找到【机器人与消息推送】，开启机器人能力。
   - 在服务器配置中填入你需要开放给钉钉回调的外网公网接口地址（对应容器的开放端口）。

## ✈️ 3. Telegram
1. **获取 Bot Token**：
   - 在 Telegram 搜索框输入：`@BotFather`，带蓝色官方认证标志的就是它。
   - 向它发送：`/newbot`。
   - 按照提示分别发送机器人的**显示名称**（Name）和**唯一用户名**（Username，必须以 `bot` 结尾）。
   - 创建成功后，BotFather 会回复一条长消息。找到中间那行 **`Use this token to access the HTTP API:`** 下面的一长串红色乱码文本（如 `12345678:AaBbCcDDEeFf...`）。
   - 这就是 **Bot Token**，将其粘贴到安装向导的 `Bot Token` 处。
2. **绑定 Chat ID (选填)**：
   - 默认情况下，如果这行**留空**，那么任何人在 Telegram 里搜到你的机器人都能跟它说话。
   - 如果你只想让它在特定群里说话，或者只和你自己单聊，你可以：在 Telegram 里拉它进群发条消息，然后在浏览器访问：`https://api.telegram.org/bot<把这里换成你的Token>/getUpdates`，在返回的一大堆 JSON 里找到 `"chat":{"id": -123123...}` 这个 ID 数字。将其粘贴到向导里。

## 🔵 4. 飞书 (Feishu/Lark)
> 和钉钉一样，单纯的群内 Webhook 只能发不能收。为了能够顺畅接收你的指令，必须创建【飞书自建应用】。

1. **进入飞书开发者后台**：访问 [open.feishu.cn](https://open.feishu.cn/)，点击右上角【开发者后台】并登录。
2. **创建企业自建应用**：点击【创建应用】 -> 【企业自建应用】，填写名称（比如 `Claw`）并确认。
3. **获取凭证**：进入应用详情页，在左侧栏【凭证与基础信息】里，找到 **`App ID`** 和 **`App Secret`**。
4. **填入脚本**：点击复制这些信息，将其填进向导的对应位置。
5. **最重要的一步（容器启动后）：开启权限与事件订阅**：
   - ⚠️ **为什么发消息机器人不回？** 因为你不给它权限，飞书就不会把你的消息转发给它。
   - 在应用后台点击【添加应用能力】，添加【机器人】。
   - 点击【权限管理】，搜索并开通以下权限：
     - `接收单聊消息` (`im:message.p2p_msg:readonly`)
     - `获取与发送单聊、群组消息` (`im:message`)
     - `接收群聊消息` (`im:message.group_msg:readonly`)
   - 点击【事件订阅】，如果是本地运行 OpenClaw，你需要配置长连接（WebSocket）或者填入服务器回调 URL，并勾选对应的消息接收事件。
   - 最后，点击【版本管理与发布】，创建一个新版本并申请发布。

## 🐧 5. QQ 
> QQ 推荐通过基于 OneBot 协议（NapCatQQ / LLOnebot等）接入，以获得完美的封禁隔离和本地通讯。这比官方 QQ 开放平台审核简单很多。

1. **准备 NapCatQQ 容器**或本地客户端（如果你用 Windows 机器挂机）。
2. **开启 Http 服务**：配置并登录 NapCatQQ，在其后台启用 `HTTP 协议` 监听。
3. **填写 Webhook URL**：
   - 通常为你 NapCatQQ 所在的局域网 IP/公网 IP + 启动端口（例如：`http://192.168.1.100:3000`）。
   - 将这串 URL 填到安装向导的 `OneBot Webhook URL` 处。
4. **填写 Token**：
   - 如果你在 NapCat 的配置里开启了 `access_token` 进行鉴权，把它填入脚本的 `Token / Secret (选填)`。如果没有开启，直接按回车留空即可。

## 🔧 6. 自定义 Webhook
如果你不属于以上任何一种软件，而是对接自己开发的系统、或是基于 Serverless 函数网关。

- **URL**：接收 JSON payload 的 HTTP/HTTPS 接口地址。
- **Token**：如果你的系统需要在请求头校验 `Authorization: Bearer <token>`，可以填。
- **上报格式**：向导让你选 JSON、Text 还是 Markdown，就是你在 webhook 另一头发版服务能识别与处理的最佳格式。通常选 `JSON`。
