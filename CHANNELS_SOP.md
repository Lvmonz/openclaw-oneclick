# 🤖 常见通讯频道机器人配置 SOP

如果你在运行 `setup.sh` 安装 OpenClaw 时选择了通讯频道，但不知道怎么获取对应的 **Webhook URL** 或 **Token / Secret**，请对照下方的图文步骤操作。

---

## 📱 1. 微信 (WeChat)
**微信不需要提前配置任何参数。**
- 在安装结束并提示容器启动后，查看日志或执行 `docker logs openclaw-main -f`。
- 终端会打印出一个“微信登录二维码”。
- 使用微信扫码授权登录即可开始对话。
- **注意**：网页版微信登录态由 `browser_profile` 挂载保留，即使之后重装 OpenClaw 也**不需要重新扫码**。

---

## 🔷 2. 钉钉 (DingTalk)
> 适用于把机器人接入某一个特定的飞书群/钉钉群聊。

1. **进入目标群聊**：在钉钉电脑端随意打开或创建一个群聊（你可以拉一个只有你和另一个人测试的群）。
2. **添加机器人**：点击群设置右上角 ⚙️ -> 找到并点击【机器人】 -> 点击【添加机器人】 -> 选择最后面蓝色的【自定义机器人】。
3. **设置名称和安全**：给机器人取随便一个名字（比如 `Claw`），**安全设置必须勾选【加签】**。
4. **获取 Secret**：勾选加签后，下方会出现一串以 `SEC...` 开头的字符。**点击复制，把它粘贴到安装向导的 `加签 Secret (选填)` 中**。
5. **获取 Webhook URL**：勾选【我已阅读并同意】点击【完成】。系统会为你生成一个 **Webhook 地址**。**点击复制，粘贴到安装向导的 `Webhook URL` 中**。

---

## ✈️ 3. Telegram
1. **获取 Bot Token**：
   - 在 Telegram 搜索框输入：`@BotFather`，带蓝色官方认证标志的就是它。
   - 向它发送：`/newbot`。
   - 按照提示分别发送机器人的**显示名称**（Name）和**唯一用户名**（Username，必须以 `bot` 结尾）。
   - 创建成功后，BotFather 会回复一条长消息。找到中间那行 **`Use this token to access the HTTP API:`** 下面的一长串红色乱码文本（如 `12345678:AaBbCcDDEeFf...`）。
   - 这就是 **Bot Token**，将其粘贴到安装向导的 `Bot Token` 处。
2. **绑定 Chat ID (选填)**：
   - 默认情况下，如果这行**留空**，那么任何人在 Telegram 里搜到你的机器人都能跟它说话。
   - 如果你只想让它在特定群里说话，或者只和你自己单聊，你可以：在 Telegram 里拉它进群发条消息，然后在浏览器访问：`https://api.telegram.org/bot<把这里换成你的Token>/getUpdates`，在返回的一大堆 JSON 里找到 `"chat":{"id": -123123...}` 这个 ID 数字。将其粘贴到向导里。

---

## 🔵 4. 飞书 (Feishu/Lark)
> 适用于把机器人接入到飞书群。

1. **进入目标群聊**：在飞书桌面端创建一个内部群。
2. **添加机器人**：点击群设置右上角 ⚙️ -> 侧边栏选择【群机器人】 -> 点击【添加机器人】 -> 选第一项【自定义机器人】。
3. **获取 Webhook**：填写机器人名称和简介，点击【添加】后，界面上会生成一条 **Webhook 地址**。这就是向导里要填的 `Webhook URL`。
4. **获取签名（Secret）**：在 Webhook 页面，勾选下方的 **【签名校验】**。会弹出一长段字符串密钥。**复制这段密钥，将其填进向导的 `签名校验 Secret` 处**。

---

## 🐧 5. QQ 
> QQ 推荐通过基于 OneBot 协议（NapCatQQ / LLOnebot等）接入，以获得完美的封禁隔离和本地通讯。这比官方 QQ 开放平台审核简单很多。

1. **准备 NapCatQQ 容器**或本地客户端（如果你用 Windows 机器挂机）。
2. **开启 Http 服务**：配置并登录 NapCatQQ，在其后台启用 `HTTP 协议` 监听。
3. **填写 Webhook URL**：
   - 通常为你 NapCatQQ 所在的局域网 IP/公网 IP + 启动端口（例如：`http://192.168.1.100:3000`）。
   - 将这串 URL 填到安装向导的 `OneBot Webhook URL` 处。
4. **填写 Token**：
   - 如果你在 NapCat 的配置里开启了 `access_token` 进行鉴权，把它填入脚本的 `Token / Secret (选填)`。如果没有开启，直接按回车留空即可。

---

## 🔧 6. 自定义 Webhook
如果你不属于以上任何一种软件，而是对接自己开发的系统、或是基于 Serverless 函数网关。

- **URL**：接收 JSON payload 的 HTTP/HTTPS 接口地址。
- **Token**：如果你的系统需要在请求头校验 `Authorization: Bearer <token>`，可以填。
- **上报格式**：向导让你选 JSON、Text 还是 Markdown，就是你在 webhook 另一头发版服务能识别与处理的最佳格式。通常选 `JSON`。
